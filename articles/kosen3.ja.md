# 小さな衛星に、推進という選択肢を載せる

KOSEN-3 CubeSat Launch Projectでは、超小型衛星に搭載するパルスプラズマスラスタの開発に関わりました。写真で見ると金属部品と配線の集合ですが、実際には「限られた体積・電力・質量の中で、宇宙で意味のある推力を出す」という制約だらけの研究でした。

![実験装置とスラスタまわり](images/kosen3_1.jpg)
*実験では、装置の状態を観察しながら測定条件を少しずつ詰めていきました。*

## 取り組んだこと

主に担当したのは、磁気ノズルによる推力向上の検討、実験データの整理、Pythonによる解析です。研究は一度測れば終わりではなく、測定条件、ノイズ、再現性、装置の癖を見ながら仮説を調整していく作業でした。

- パルスプラズマスラスタの性能評価
- 磁場を使った推力向上の検討
- 測定データのPython解析
- React + TypeScriptによる発表資料の作成
- 国際学会での発表と論文執筆

![研究発表や資料作成の様子](images/kosen3_2.jpg)
*数式やグラフだけでなく、相手に伝わる形にするところまでが研究だと感じました。*

## 学んだこと

この研究で一番鍛えられたのは、曖昧な現象を扱う姿勢です。理論上はこうなるはず、という予想と、実験で出てくる値は簡単には一致しません。その差分を雑に片付けず、装置、測定、解析、仮説のどこに原因があるかを切り分ける必要がありました。

> 面白い現象を見つけることより、再現できる形で説明することの方が難しい。

今のソフトウェア開発でも、この経験はかなり効いています。ログを見る、条件を揃える、仮説を小さく検証する。研究で身についた地味な手つきが、そのまま実装やデバッグに繋がっています。
