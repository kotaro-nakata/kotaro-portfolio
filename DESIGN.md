# Design System & Log

Kotaro Nakata ポートフォリオのデザインルールと変更履歴をまとめたドキュメント。
新しいセクションやコンポーネントを足すときは、このルールに沿うことで統一感を保つ。

---

## 1. デザイン原則

- **統一感** — サイズ・余白・角丸・影は決められたトークン（CSS変数）から選ぶ。場当たり的な数値を書かない。
- **親しみやすさ** — やわらかい角丸・淡い拡散影・たっぷりの余白・あたたかいグラデーション背景。
- **継ぎ目をつくらない** — 背景は一枚の連続したグラデーション。セクションごとの帯は使わず、必要ならフェードするティントで差をつける。
- **可読性優先** — 極端に小さい文字をつくらない（最小 `--fs-xs` = 0.8rem）。本文 16.5px / 行間 1.75。
- **アクセシビリティ** — `prefers-reduced-motion` を尊重。インタラクティブ要素には `aria-*` を付ける。

## 3. タイポグラフィ

**一本の音階（type scale）に全見出しをぶら下げる。** 個別に `font-size` を直書きしない。

| 変数 | 値 | 用途 |
|---|---|---|
| `--fs-hero` | `clamp(2.2rem, 5.2vw, 3.3rem)` | ヒーロー H1 |
| `--fs-h2` | `clamp(1.45rem, 3vw, 1.95rem)` | セクション見出し |
| `--fs-h3` | `1.02rem` | カード見出し |
| `--fs-lead` | `clamp(1.05rem, 2vw, 1.3rem)` | リード文・キャッチ |
| `--fs-body` | `1rem` | 本文 |
| `--fs-sm` | `0.9rem` | カード説明・リスト |
| `--fs-xs` | `0.8rem` | チップ・日付・タグ（最小） |
| `--fs-eyebrow` | `0.78rem` | アイブロウ（大文字ラベル） |

- フォント: `"Inter", "Noto Sans JP", system-ui, sans-serif`
- 見出しウェイト 800 / 本文 400 / ラベル・タグ 700
- ヒーロー・キャッチは**中央寄せ**（`.hero-minimal { text-align: center }`）
- セクション見出しは単語1つで統一: **Works / Projects / Skills / Experience**
  （アイブロウ＋サブタイトルの2段組みは使わない）

## 4. 背景（動的・遊び心）

- 背景は3つのソフトな光球（緑・オレンジ・ミント）を `radial-gradient` で重ねる。
- 位置は CSS変数 `--gx/--gy`・`--ax/--ay`・`--bx/--by`(%) で制御し、`main.js` が毎フレーム更新。
- **2層の動き**:
  1. 自律ドリフト — 各球が別々の周期・位相の sin/cos（リサージュ曲線）でゆっくり循環。静止時も動く。
  2. カーソル追従 — マウス位置へイージングで寄る視差。緑/オレンジは逆方向、ミントは控えめ。
- `background-attachment: fixed` でスクロールしても継ぎ目が出ない。
- 透明度の目安: 緑 `.28` / オレンジ `.30` / ミント `.22`（濃さ調整はここ）。
- `prefers-reduced-motion: reduce` のときはアニメを止めて静止配置。

---

## 5. コンポーネント規約

- **カード**（`.work-card` / `.aside-item` / `.skill-card`）: `--card`面 + `1px var(--border)` + `--radius` + `--shadow`。ホバーで `translateY(-4px)` + `--shadow-hover`。
- **チップ／タグ**: `border-radius: 999px`、文字 `--fs-xs` / weight 700。塗り(`solid`)は緑地に白、枠(`outline`)はミュート色。
- **ボタン**: `border-radius: 999px`、`min-height: 44px`（タップ領域確保）。`primary`=緑塗り / `ghost`=枠線。
- **モーダル/詳細ビュー**: 角丸 `--radius`、スマホでは画面下から出るシート型。
- **ナビ**: デスクトップは横並び。**820px以下でハンバーガー**に切替（下ドロップ式・ブラー背景・×印アニメ）。リンクタップ/外側クリック/Escで閉じる。`aria-expanded` 連動。

---

## 6. レスポンシブ

- `clamp()` による無段階スケーリング + メディアクエリの併用。
- ブレークポイント:
  - **≤820px** — ナビをハンバーガー化、Skills を縦リスト型に再構成、Contact を1カラム。
  - **≤640px** — カードグリッドの最小幅を縮小、間隔を詰める。
  - **≤560px** — 全グリッド1カラム、セクション余白縮小、モーダルをシート型に。
- `<meta name="viewport">` 設定済み。画像は `max-width: 100%`。グリッドは `auto-fit` で枚数自動折返し。

## 7. 変更履歴（Log）

### 2026-06-13 — デザイン全面リフレッシュ
- **タイポ統一**: バラバラだった文字サイズ（hero 6.4rem〜chip 0.72rem）を一本の type scale に集約。極小文字を底上げ、巨大見出しを抑制。最終的に hero `clamp(2.2,5.2vw,3.3rem)` / h2 `clamp(1.45,3vw,1.95rem)` に調整。
- **背景の継ぎ目を解消**: `.section.alt` の不透明バンド（固定背景上をスクロールして線が見えていた）を撤去。上下フェードのティントに置換し、背景を一枚に統一。
- **親しみやすさ**: 角丸 14→18px、影をやわらかい拡散影に統一、カード余白拡大、ヘッダー/背景トーンを調整。
- **動的背景を追加**: 3つの光球が自律ドリフト＋カーソル追従。`prefers-reduced-motion` 対応。死んでいた旧マウス追従コードを置換。視認性向上のため濃さと追従幅を強化。
- **見出し整理**: Skills のサブタイトル「宇宙からWebまで〜」を削除し「Skills」のみに。Story の「Ex & Story / 好奇心の履歴。」を「Experience」に統一（4セクション見出しを単語1つで揃えた）。
- **ヒーロー中央寄せ**: 「Kotaro Nakata」＋キャッチを中央配置に。
- **ハンバーガーメニュー**: 820px以下のモバイルナビを実装（ドロップダウン・×印アニメ・自動クローズ・aria対応）。

### 2026-06-13 — 横スライダー & コピー変更
- **Works / Projects / Aside を横スライダー化**: `scroll-snap` + ドット + 次カードのチラ見せ + ドラッグスクロール（4px超で「ドラッグ」判定しモーダル誤爆を抑制）。矢印・自動再生は無し。
  - 表示枚数（カードサイズ）はデスクトップ3枚 `/3`・タブレット約2枚 `/2.2`・スマホ約1枚 `/1.12`。デプロイ済みのグリッドと同等サイズ。
  - **ドットは「はみ出して初めて表示」**: 件数が画面に収まる間（例: 3件）は非表示で、項目が増えてスクロールが生じると自動で出現。
- **ヒーローのキャッチ変更**: 「あなたの『やりたい』と『困った』に、テックと対話でコミットする。」
